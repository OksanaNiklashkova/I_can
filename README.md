# I_can

Этот проект предоставляет возможности для разворачивания веб-приложения по созданию у пользователя полезных привычек.

## Описание
В 2018 году Джеймс Клир написал книгу «Атомные привычки», которая посвящена приобретению новых полезных привычек и искоренению старых плохих привычек.
В проекте представлено веб-приложение-трекер, цель которого - формирование у человека полезных привычек. Пользователь, зарегистрировавшись в приложении "I_can" может создавать привычки двух видов: приятные и полезные.
Приятная привычка имеет ограниченный набор атрибутов: действие, дату создания, (опционально) место и признак приятной привычки.
Полезная привычка может обладать следующими атрибутами: действие, место, время начала, время исполнения, периодичность, связанную с ней приятную привычку или вознаграждение, флаг отправки уведомления в Телеграм и дата последней отправки уведомления.
Оба вида привычек при создании закрепляются за пользователем, изменять и удалять их может только создатель.
У привычек есть признак публичности - если привычка публичная, то ее могут видеть в списке другие пользователи.

## Точка входа
Для перехода в веб-приложение запустите команду `python manage.py runserver`  и нажмите на ссылку в консоли `http://127.0.0.1:8000/` 
Ссылка для регистрации пользователя - `http://127.0.0.1:8000/register/`
Ссылка для входа по логину - `http://127.0.0.1:8000/login/`


## Тестирование

В рамках проекта реализовано тестирование всех эндпойнтов с помощью unittest. 
Отчет о покрытии находится в файле htmlcov_index.html в корне проекта.

## Установка

Для установки и запуска проекта необходимо выполнить следующие шаги:

1.  **Клонируйте репозиторий:**

    ```
    https://github.com/OksanaNiklashkova/I_can.git
    ```

2.  **Перейдите в папку проекта:**

    ```
    cd I_can
    ```

3.  **Установите зависимости с помощью Poetry:**

    ```
    poetry install
    ```
    
4.  **Создайте базу данных и пропишите настройки для доступа к ней в файле .env (образец заполнения в файле .env.sample)**

5.  **Запустите команду запуска сервера, чтобы перейти в веб-приложение**
    ```
    python manage.py runserver
    ```

## Использование

Для начального заполнения базы данных воспользуйтесь фикстурами
~~~
python manage.py loaddata users_fixture.json --format json
python manage.py loaddata habits_fixture.json --format json
~~~
При указании у пользователя chat_id от Telegram (10 цифр), при помощи периодических задач Celery реализован функционал отправки напоминаний пользователю о его полезных привычках.
С периодичностью 1 раз в час в период с 9 утра до 21 часа происходит проверка - наступило ли время начала (start_time) полезной привычки. Если текущее время больше времени начала - пользователю в телеграм отправляется сообщение.
При этом параметру `update_reminder_send_flag` присваивается новое значение (текущая дата), а флаг в артибуте `update_reminder_send_flag` меняется на True.
Раз в сутки, в 00:00 запускается периодическая задача, проверяющая привычки с флагом update_reminder_send_flag=True и если разница между датой последней отправки напоминания и текущей датой равна значению атрибута `periodicity` (периодичность), флаг в `is_reminder_send` меняется на False.
Таким образом, при следующем запуске периодической задачи `send_reminder_to_telegram` все привычки, чье время наступило, попадают в выборку для отправки напоминаний.
